(1) HMAC Secret Storage

Question: How can we securely store HMAC-SHA256 secrets for request signing in a client-side environment without exposing them?

My Understanding: Each client needs a secret key to sign requests, but a static secret in the source code is insecure.

Solution: Implement a Session-Specific Secret generated by the server after a successful login handshake, stored only in memory (Pinia/Vuex) and rotated per session.

(2) Encryption Key Rotation

Question: How do we rotate keys every 180 days without the performance hit of re-encrypting the entire database?

My Understanding: Data encrypted with old keys must remain accessible after the rotation period.

Solution: Implement Key Versioning by storing a key_id alongside encrypted blobs and maintaining a "Key Vault" of historical keys to avoid massive batch re-encryption.

(3) Face Deduplication Scope

Question: Should the deduplication check against a local similarity threshold include inactive members?

My Understanding: We only need to check against current, active users to ensure uniqueness.

Solution: The check must include Deactivated accounts to prevent banned or former members from re-enrolling as new users to bypass fees or restrictions.

(4) PIN Re-entry vs. Session Expiry

Question: How does the optional PIN interact with the standard 30-minute session expiry?

My Understanding: The PIN is a convenience feature for "warm" workstations.

Solution: Maintain a Full Session timer (30 mins) and a separate PIN Lock Timer (5 mins) that triggers a UI overlay. If the full session expires, the PIN is invalidated and a password is required.

(5) Liveness Challenge Validation

Question: What constitutes a successful validation of the two-step head-turn challenge?

My Understanding: The system simply captures two frames to confirm movement.

Solution: The backend must validate that biometric landmarks (like nose tip and eye alignment) have shifted significantly and in the correct direction to prevent spoofing with a static photo.

(6) Offline Shared Folder Path

Question: How will the Express backend interface with the designated "offline" shared folder for reports?

My Understanding: The server saves files to a standard network drive.

Solution: Define a configurable SMB/UNC path mapping in the .env file and ensure the Express service account has explicit write permissions for that specific volume.

(7) Dashboard Builder Permissions

Question: How can we allow admins to use a drag-and-drop dashboard builder without overwriting each other's layouts?

My Understanding: Admins can globally move and configure widgets.

Solution: Implement a Layout Metadata Schema in MySQL that stores a JSON blob of coordinates and ChartIDs per user/role, allowing for personalized views.

(8) Face Capture Quality Gates

Question: Where should the "Quality Gate" feedback (blur, positioning) be processed to ensure immediate feedback?

My Understanding: The user needs to know if their photo is acceptable in real-time.

Solution: Process Initial Gates (blur, face-in-frame) in the browser via TensorFlow.js for speed, while the Final Gate (similarity/deduplication) is handled by the backend for security.

(9) Content View Attribution

Question: How do we track content views by source channel (e.g., Kiosk vs. Terminal) if the user isn't logged in?

My Understanding: We need to track which station the data is being viewed from.

Solution: Require the frontend to pass a DeviceID or StationToken in the analytics API headers to differentiate between various physical hardware locations.

(10) Audit Trail Immutability

Question: How do we guarantee the audit trail remains immutable for the required 7-year retention period?

My Understanding: Records are simply marked as "do not delete" in the application logic.

Solution: Implement a Database Trigger that explicitly blocks UPDATE and DELETE commands on the audit_logs table and triggers a system integrity alert if attempted.

(11) Role Hierarchy and Navigation

Question: How should the navigation and permissions be structured between Coaches and Admins?

My Understanding: Coaches see a subset of features compared to Admins.

Solution: Use Inclusive Permissions where higher roles inherit all permissions of lower roles, using a v-if directive to filter navigation links based on the user's hierarchy level.

(12) Deactivation vs. Search Trends

Question: How should deactivated member data affect analytics and search trend reports?

My Understanding: Deactivated members are simply flagged as inactive in the database.

Solution: Analytics queries must filter for active status by default to avoid skewing performance metrics; historical data should only be included via an explicit toggle.

(13) API Rate Limiting vs. Kiosks

Question: How do we prevent a 60 RPM limit from blocking multiple kiosks sharing a single LAN IP?

My Understanding: A standard IP-based rate limit will protect the server.

Solution: Implement Identity-Based Rate Limiting, allowing higher thresholds for authenticated stations and users while keeping the unauthenticated "Sign-in" page strictly capped.

(14) Subscription Delivery Failure

Question: What happens if a report subscription fails to deliver to the shared folder due to a network error?

My Understanding: The report is generated and the system tries again later.

Solution: Use a Background Worker (Cron) to deliver the report to an "In-App Inbox" regardless of folder availability, flagging a "Storage Sync Error" for admins to investigate.

(15) Restore Workflow Endpoint

Question: What is the intended target and validation process for the "Restore Workflow" endpoint?

My Understanding: The system needs a way to recover from data corruption or failure.

Solution: Target a Standby Recovery Instance using a "Dry Run" mode that validates checksums and decryption keys before performing any overwrite on the production MySQL schema.
